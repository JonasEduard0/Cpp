		Estruturas de dados
Ao contrário da Programação Estruturada, gira em torno de classes e não funções, baseado na análise bottom-up e não na top-down.
Herança: transmissão de recursos de uma superclasse/classe base pra classe filha/subclasse.

Classe: é o molde que serve de base para criar/instanciar objetos. Ex: receita ou planta de construção.
Objeto: é instanciado/criado a partir da classe. Ex: bolo ou casa construída.
Atributos(Propriedades): são as variáveis que caracterizam um objeto.
Métodos(Funções): os comportamentos do objeto, como ele funciona.

	Red-black tree: quando há desbalanceamento, tem q ver a posição e cor do TIO do elemento q desbalanceou pra decidir qual correção usar.
1ª Todo nó é vermelho ou preto, 2ª raiz é preta, 3ª Todo nó nulo(Nil/null) é preta, 4ª nó vermelho, filhos só pretos, 5ª Pra cada nó, todos caminhos do nó pra folhas descendentes tem o mesmo número de nós PRETOS.

	Hash
Se 2+ chaves produzirem o mesmo hash code, é uma colisão. Neste caso o Hash armazena esses pares no mesmo bucket como uma lista encadeada ou outra.
M = mod(modulo), resto de uma divisao. Deve ser o tamanho da lista pra caber na tabela. Prefira primos.
Nem sempre “mais contas” significam menos colisões. É preciso testar e medir.
Não existe função perfeita. O melhor depende dos seus dados. O importante é experimentar, contar colisões e comparar.

 Divisão
Método mais simples. Pegamos a chave e fazemos o resto da divisão pelo tamanho da tabela:
h(k) = k mod M.

 Hash para string
Transformar letra em numero (A=1, B=2, …) e somar. Esse somatório é então reduzido pelo tamanho da tabela (mod M).
Podemos criar variações: multiplicar cada letra por um fator, acumular com base (hash = (hash*37 + letra) mod M).

 Meio do quadrado
Embaralhar mais os dígitos da chave. O processo é:
1. Eleva k ao quadrado; 2. Pegar os dígitos centrais do resultado; 3. Aplicar mod M.
Exemplo: k = 3121 → k² = 9740641. Pegamos o miolo 406 → 406 mod 10 = 6. Portanto, a chave 3121 vai para a posição 6. Pode misturar os dígitos e pra espalhar melhor.

 Extração
Neste método, em vez de usar a chave inteira, escolhemos apenas uma parte dela.
Exemplo: uma matrícula universitária 2014.22.123. os primeiros dígitos sao o ano e, mas os 3 últimos diferenciam os alunos. Podemos então usar 123 mod M para decidir a posição.

 Dobramento
Cortar chave em blocos de mesmo tamanho, somá-los e aplicar M.
Exemplo: 13320020 → quebramos em blocos 133, 200 e 20. Somamos: 133 + 200 + 20 = 353. Depois aplicamos 353 mod 10 = 3. A chave vai para a posição 3.
Variação: no dobramento em sanfona, invertemos alguns blocos antes de somar, para misturar mais os dígitos.
Prós: aproveita todos os dígitos da chave. Cuidados: precisa escolher blocos de tamanho equilibrado.

  Tratamento de colisões
 Endereçamento fechado
Cada bucket guarda uma lista encadeada com as chaves daquele índice.
Prós: simples; sem “estouro” por bucket(limite é a memória). 
Contras: se o hash espalha mal, algumas listas ficam grandes.

 Endereçamento aberto/linear
Usamos apenas o vetor; em colisão, procuramos a próxima posição livre (busca linear circular), tem 3 repositorios por posiçao.
Prós: estrutura contígua (bom de cache), sem alocação de nós.
Contras: pode gerar clusters (aglomerados) e exigir rehash com mais frequência.


		Big-O Notation e Complexidade de algoritmos
Forma de descrever como o custo(tempo/memória) de um algoritmo cresce quando a entrada aumenta. 
Mede o número de operações fundamentais. n = tamanho de entrada.

	Regras
1) Ignorar contante: O(1000n) vira O(n)
2) Ignorar termos menores: O(n log n + n) vira O(n log n)
3) Pior caso: um bloco custa O(n) e outro O(n²), então O(n) + O(n²) = O(n²). O pior caso domina.
4) Laços aninhados multiplicam: O(n) * O(n) = O(n²)
5) Entradas diferentes geram variáveis diferentes: O(n + m) ou O(n*m)

Como calcular Big-O de um código: 1-Descobrir o que depende do tamanho da entrada. 
  2-Ver quantas vezes a operação principal acontece. 3-Eliminar constantes e termos menores.

Principais Big-O:
  O(1):     Tempo não cresce com n, constante. Você não percorre, vai direto. Ex: Acessar posição por índice.
  O(log n): A cada passo, reduz o problema pela metade. Ex: busca binária.
  O(n):	    Trabalho cresce proporcional com entrada, passa por todos elementos uma vez. Ex: percorrer vetor.
  O(n log n): Comum em algoritmos eficientes de ordenação como Heap Sort.
  O(n²):      Normalmente dois laços aninhados. Ex: Bubble Sort.
  O(2^n):     Cada aumento pequeno em n faz o custo explodir. Ex: Fibonacci, cada decisão gera várias outras.
  O(n!):      Quando testamos todas permutações/ordens possíveis.

	
	Modelo RAM(random access machine)
Cada operação simples(a+b=c;) conta como 1 passo, laços e sub-rotinas são compostos por várias operações simples e cada acesso à memória também conta como 1 passo. Medimos quantas ações básicas o código faz.
Focamos no pior caso, pois diz o máximo que o algoritmo pode custar, isso é mais seguro para comparar algoritmos.

Pior caso = maior número de passos para entradas de tamanho n
Melhor caso = menor número de passos
Caso médio = média entre as entradas possíveis daquele tamanho.

Complexidade exata: contar tudo detalhadamente, difícil pois depende de muitos detalhes e de casos particulares.
Complexidade assintótica: olha o crescimento quando n fica grande, mais prático. Expressa o limite do crescimento.


	Limites
Limite superior: O(g(n)). O algoritmo não cresce mais rápido que isso, para n suficientemente grande.
  Ex: 3n² - 100n + 6 é O(n²), também é O(n³), porque n³ cresce ainda mais rápido.

Limite inferior: Ω(g(n)). Cresce pelo menos nessa ordem. 
  Ex: 3n² - 100n + 6 é Ω(n²), também é Ω(n), porque ele cresce pelo menos linearmente.

Limite superior e inferior: Θ(g(n)). Exatamente a ordem de crescimento.
  Ex: 3n² - 100n + 6 é Θ(n²), pois cresce nem menos nem mais que n². 

n² = O(n³). Quer dizer que n³ é um teto válido para n². Mas n²=Θ(n³) é falso, ñ é o crescimento exato de n².

	Constante n0 
Um algoritmo tem um comportamento meio estranho para n = 1, 2,... mas depois de n = 100 ele cresce como n².
Esse “a partir daqui” é o papel do n0.

	Complexidade espacial.
Mede a memória necessária para executar o algoritmo. Considera tanto o tamanho da entrada quanto o espaço auxiliar de execução. Tempo = quanto trabalha, Espaço = quanta memória extra usa.

	Estruturas
Árvore balanceada: Sempre O(log n) se balanceada.
Vetor dinâmico:    Acesso = O(1); Percorrer = O(n); Insert  = O(1/n); Remoção = O(n); Busca = O(log n).

Hash:  		   Acesso =  -;   Percorrer = O(n); Insert  = O(1);   Remoção = O(1); Busca = O(1).

	Ordenação
Bubble Sort: 	O(n²). Compara elementos vizinhos e troca. ## Recursos extras

Selection Sort: O(n²). Escolher o menor elemento e colocar na posição correta.

Insertion Sort: O(n²). Inserir cada elemento na posição correta. Na prática é mais rápido que os outros.

Fazem este cálculo: n + (n-1) + (n-2) + ... + 1 -> n(n+1)/2 = O(n²). 
Se problema tem tamanho n < 10.000 ainda são úteis. Problema é maior em algoritmos n ≥ 100.000.

Merge Sort e Quick Sort: O(n log n). Usado em sistemas reais.
